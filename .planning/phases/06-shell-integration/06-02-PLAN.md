**Create shell-integration/install.sh:**

```bash
#!/bin/bash
# Terminal Remote Shell Integration Installer
# Installs init scripts to ~/.terminal-remote/

set -e

INSTALL_DIR="$HOME/.terminal-remote"
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"

echo "Installing Terminal Remote shell integration..."

# Create installation directory
mkdir -p "$INSTALL_DIR"

# Copy init scripts
cp "$SCRIPT_DIR/init.zsh" "$INSTALL_DIR/"
cp "$SCRIPT_DIR/init.bash" "$INSTALL_DIR/"
cp "$SCRIPT_DIR/init.fish" "$INSTALL_DIR/"

echo "Installed to $INSTALL_DIR/"
echo ""
echo "Add to your shell configuration:"
echo ""
echo "  Zsh (~/.zshrc):"
echo "    source ~/.terminal-remote/init.zsh"
echo ""
echo "  Bash (~/.bashrc):"
echo "    source ~/.terminal-remote/init.bash"
echo ""
echo "  Fish (~/.config/fish/config.fish):"
echo "    source ~/.terminal-remote/init.fish"
echo ""
echo "Note: Add the source line at the END of your rc file"
echo "      (after oh-my-zsh, starship, or other prompt tools)"
```

Make executable: `chmod +x shell-integration/install.sh`

<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Shell integration scripts that connect terminal sessions to mac-client with:
1. Auto-connect on shell startup

3. Disconnect warning when mac-client quits
4. Auto-reconnect when mac-client restarts

  </what-built>
  <how-to-verify>
**Test 1: Session appears in mac-client**

1. Start mac-client:
   ```bash
   cd mac-client && cargo run
   ```
   (Leave running in background or a separate terminal)

2. Open a NEW terminal window/tab (so shell sources the init script)
   - Should print "Connected to Terminal Remote"

3. Check mac-client menu bar:
   - Click the tray icon
   - Should show "1 session" or session count

**Test 2: Session name is correct**

1. In the new terminal, check PWD:
   ```bash
   pwd  # Note the directory name
   echo $$  # Note the PID
   ```

2. Session name in mac-client logs should be "dirname [PID]"
   (Check terminal running cargo run for log output)

**Test 3: Live directory tracking (CRITICAL)**

1. In the terminal with integration, change directory:
   ```bash
   cd /tmp
   ```

2. Check mac-client logs - should see session rename to "tmp [PID]"

3. Change directory again:
   ```bash
   cd ~
   ```

4. Check mac-client logs - should see session rename to home dir name

**Test 4: Disconnect warning**

1. Keep the integrated terminal open
2. Stop mac-client (Ctrl+C in the cargo run terminal)
3. In the integrated terminal, press Enter (triggers prompt)
4. Should see "Terminal Remote disconnected" message (once)

**Test 5: Auto-reconnect**

1. With integrated terminal still open (showing disconnected)
2. Start mac-client again:
   ```bash
   cd mac-client && cargo run
   ```
3. In the integrated terminal, press Enter (triggers prompt check)
4. Should see "Connected to Terminal Remote" message
5. Mac-client should show the session again

**Test 6: Clean disconnect on exit**

1. In the terminal with integration, type `exit`
2. Mac-client should show session disconnected in logs
3. Session count should decrease

**Test 7: Startup without mac-client**

1. Stop mac-client (Ctrl+C)
2. Open a NEW terminal
3. Should see "Terminal Remote not running" message
4. Startup should feel instant (no delay)
5. Shell works normally

**Test 8: Works with your prompt theme**

1. If you use starship, p10k, or oh-my-zsh, verify:
   - Prompt still looks correct
   - No error messages about hooks
   - No perceptible delay
  </how-to-verify>
